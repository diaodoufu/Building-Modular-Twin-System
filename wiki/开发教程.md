# 开发教程

## 开发环境搭建

详见 [[部署指南]]，这里补充开发相关的额外配置。

### 推荐 IDE 配置

**VS Code 推荐插件**：
- Vue - Official（Vue 3 语法支持）
- Python（Python 语言支持）
- Pylance（Python 类型检查）
- ESLint（代码规范）

### 前端开发

```bash
cd bmts-frontend
npm install
npx vite --host    # 启动开发服务器，支持热更新
```

### 后端开发

```bash
cd bmts-backend
source venv/bin/activate    # Windows: venv\Scripts\activate
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

`--reload` 参数会自动检测代码变更并重启服务。

## 项目结构

```
BMTS/
├── bmts-frontend/          # 前端项目
│   └── src/
│       ├── api/            # API 调用封装（axios）
│       ├── components/     # 可复用组件
│       ├── composables/    # Vue Composition API
│       ├── pages/          # 页面组件
│       ├── router/         # 路由配置
│       └── stores/         # Pinia 状态管理
├── bmts-backend/           # 后端项目
│   └── app/
│       ├── api/            # API 路由（Controller 层）
│       ├── models/         # SQLAlchemy 数据模型
│       ├── schemas/        # Pydantic 模型 + BMTS Schema
│       ├── services/       # 业务逻辑（Service 层）
│       └── utils/          # 工具函数
└── docs/                   # 项目文档
```

## 开发流程

### 添加新 API

1. **定义模型**（如需要新表）：在 `app/models/models.py` 中添加 SQLAlchemy 模型
2. **定义 Schema**：在 `app/schemas/schemas.py` 中添加 Pydantic 请求/响应模型
3. **实现业务逻辑**：在 `app/services/` 中添加 service 函数
4. **创建路由**：在 `app/api/` 中添加路由，调用 service 层
5. **注册路由**：在 `app/main.py` 中 include_router

### 添加新页面

1. **创建页面组件**：在 `src/pages/` 中创建 Vue 组件
2. **配置路由**：在 `src/router/index.ts` 中添加路由定义
3. **添加导航入口**：在相关页面添加跳转按钮/链接
4. **封装 API**：在 `src/api/` 中添加对应的 API 调用函数

### 添加 3D 功能

BMTS 的 3D 功能通过 `useThreeScene` composable 管理：

```typescript
// 在 Vue 组件中使用
import { useThreeScene } from '../composables/useThreeScene'

const { scene, camera, renderer, addObject, removeObject } = useThreeScene(containerRef)
```

Three.js 相关代码集中在 `composables/useThreeScene.ts`，保持 3D 逻辑与 Vue 组件的解耦。

## 代码规范

### 提交信息

使用 Conventional Commits：
- `feat: 新功能`
- `fix: 修复 Bug`
- `docs: 文档更新`
- `refactor: 代码重构`

### 前端规范

- Composition API + `<script setup>`
- TypeScript 严格模式
- API 调用封装在 `src/api/`，不直接在组件中写 axios

### 后端规范

- PEP 8 代码风格
- 使用 type hints
- API → Service → Model 三层架构
- 路由只做参数校验和调用 service，业务逻辑在 service 层

## 常见问题

### 时区问题

前端传入的时间视为 CST（UTC+8），后端在存储前会自动添加时区信息并转换为 UTC。请勿在前端手动添加时区偏移。

### 数据库迁移

目前使用 `init_db()` 自动建表方式。未来会引入 Alembic 做正式的数据库迁移管理。

### 3D 场景性能

如果房间数量较多（>100），建议使用 LOD（细节层次）策略，远处房间合并渲染，近处才显示细节。
