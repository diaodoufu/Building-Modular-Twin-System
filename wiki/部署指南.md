# 部署指南

## 环境要求

| 依赖 | 最低版本 | 推荐版本 |
|------|----------|----------|
| Python | 3.11 | 3.12 |
| Node.js | 18 | 20 LTS |
| PostgreSQL | 14 | 16 |
| Git | 2.x | 最新 |

## 开发环境部署

### 1. 克隆仓库

```bash
git clone https://github.com/diaodoufu/Building-Modular-Twin-System.git
cd Building-Modular-Twin-System
```

> **提示**：更详细的部署指南请参考项目根目录的 [DEPLOYMENT.md](../DEPLOYMENT.md)，包含 Windows/macOS/Linux 三平台的完整步骤和常见问题排查。

### 2. 配置 PostgreSQL

```bash
# 创建数据库
psql -U postgres
CREATE DATABASE bmts;
\q
```

### 3. 后端部署

```bash
cd bmts-backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env，填入实际配置：
# DATABASE_URL=postgresql+asyncpg://postgres:your_password@localhost:5432/bmts
# SECRET_KEY=your-secret-key
# DEBUG=true

# 初始化数据库（建表 + 示例数据）
python seed.py

# 启动开发服务器
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端 API 文档：http://localhost:8000/docs

### 4. 前端部署

```bash
cd bmts-frontend

# 安装依赖
npm install

# 启动开发服务器
npx vite --host
```

前端访问地址：http://localhost:3000

### 5. 验证部署

1. 访问 http://localhost:3000，应看到首页
2. 使用测试账号登录：
   - 管理员：admin / admin123
   - 普通用户：testuser / test123
   - 访客：viewer / viewer123
3. 点击建筑卡片进入 3D 视图

## 生产环境部署

> **提示**：完整的生产环境部署指南请参考项目根目录的 [DEPLOYMENT.md](../DEPLOYMENT.md)。

### 前端构建

```bash
# 构建生产版本
cd bmts-frontend
npm run build

# dist/ 目录可部署到 Nginx 或任何静态文件服务器
```

## 环境变量说明

### 后端 (.env)

| 变量 | 说明 | 默认值 |
|------|------|--------|
| DATABASE_URL | PostgreSQL 连接字符串 | postgresql+asyncpg://postgres:postgres@localhost:5432/bmts |
| SECRET_KEY | JWT 签名密钥 | bmts-dev-secret-key-change-in-production |
| DEBUG | 调试模式 | true |

**重要**：生产环境务必修改 `SECRET_KEY` 和 `DATABASE_URL`。
