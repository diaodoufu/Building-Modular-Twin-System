# 权限体系说明

## 概述

BMTS 实现三级权限体系，确保不同角色的用户只能执行其权限范围内的操作。

## 角色定义

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| **admin（管理员）** | 组织管理员 | 全部权限，含数据管理、预约审核、用户管理 |
| **user（组织用户）** | 组织内普通成员 | 查看空间、创建/管理自己的预约 |
| **viewer（访客）** | 未登录或只读用户 | 仅查看空间信息，无任何写操作 |

## 权限矩阵

| 操作 | admin | user | viewer |
|------|-------|------|--------|
| 查看空间信息 | ✅ | ✅ | ✅ |
| 查看房间状态 | ✅ | ✅ | ✅ |
| 查看房间日程 | ✅ | ✅ | ✅ |
| 创建预约 | ✅ | ✅ | ❌ |
| 取消自己的预约 | ✅ | ✅ | ❌ |
| 取消他人的预约 | ✅ | ❌ | ❌ |
| 审核预约（通过/拒绝） | ✅ | ❌ | ❌ |
| 创建容器（建筑/楼层/房间） | ✅ | ❌ | ❌ |
| 编辑容器属性 | ✅ | ❌ | ❌ |
| 删除容器 | ✅ | ❌ | ❌ |
| 导入/导出数据 | ✅ | ❌ | ❌ |
| 查看统计数据 | ✅ | ✅ | ❌ |
| 访问管理后台 | ✅ | ❌ | ❌ |

## 前端路由守卫

### requiresAuth

需要登录才能访问的页面。未登录用户会被重定向到登录页。

### requiresAdmin

需要管理员权限才能访问的页面。非管理员用户会被重定向到首页。

受保护的路由：
- `/admin` — 管理后台
- `/review` — 预约审核

## 后端权限校验

### JWT Token 校验

所有写操作需要在请求头中携带 Bearer Token：

```
Authorization: Bearer eyJ...
```

### 组织归属校验

预约相关操作（创建/更新/取消/审核）会校验用户是否属于该预约对应的组织：

```python
# 校验用户是否为指定组织的管理员
async def _check_org_admin(user_id: str, org_id: str):
    member = await get_org_member(user_id, org_id)
    if not member or member.role not in ("owner", "admin"):
        raise HTTPException(403, "需要组织管理员权限")

# 校验用户是否为指定组织的成员
async def _check_org_member(user_id: str, org_id: str):
    member = await get_org_member(user_id, org_id)
    if not member:
        raise HTTPException(403, "需要组织成员权限")
```

### 预约操作权限

- **创建预约**：需要是该房间所属组织的成员
- **取消预约**：仅预约本人或组织管理员
- **审核预约**：仅组织管理员
- **预约状态限制**：只能取消状态为 `pending` 或 `approved` 的预约

## Phase 6 扩展：容器级权限

Phase 6 将实现更细粒度的容器级权限，允许管理员授权用户只能查看/操作指定容器（某栋楼/某层/某房间）：

```sql
CREATE TABLE container_permissions (
    id          UUID PRIMARY KEY,
    org_id      UUID NOT NULL,
    user_id     UUID NOT NULL,
    container_id UUID NOT NULL,
    permission  VARCHAR(16) NOT NULL,  -- view|edit|manage
    UNIQUE(org_id, user_id, container_id)
);
```
